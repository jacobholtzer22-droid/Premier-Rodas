# Photos & videos

Real job photos and videos are live across the whole site. This documents how they're wired up
so they're easy to update later.

## Photos

- **Source:** Clay's 175 job photos, optimized to **1400px long edge, ~360 KB each** (62 MB total)
  via macOS `sips`. Raw originals were moved out of the repo to
  `~/Desktop/Rodas-Website-Photos-ORIGINALS` (never committed).
- **Location:** `public/images/gallery/rodas-001.jpg … rodas-175.jpg`
- **Manifest:** `lib/gallery.ts` (`GALLERY_IMAGES`) — auto-generated by listing the directory.
  The `/gallery` page maps over it with a lazy `next/image` grid + "Load more" (12 at a time).

### Regenerating the manifest after adding/removing photos

```bash
cd "Rodas Premier Lawn & Snow Services Website"
{ echo "// AUTO-GENERATED — do not edit by hand."; echo "export const GALLERY_IMAGES: string[] = ["; \
  for f in $(ls public/images/gallery/*.jpg | sort); do echo "  '/images/gallery/$(basename "$f")',"; done; \
  echo "];"; echo ""; echo "export const GALLERY_COUNT = GALLERY_IMAGES.length;"; } > lib/gallery.ts
```

## Videos

- **Note:** Of the 155 source `.mov` files, 105 were **HEVC** (won't play in Chrome/Firefox) and
  ~6.9 GB total — too large to ship. We selected **6 short, web-safe H.264 clips** (portrait,
  720×1280, 3–9 s, ~25 MB total) and copied them to `public/videos/clip-01.mp4 … clip-06.mp4`.
- **Where they show:** the **"See us in action"** section (`components/VideoShowcase.tsx`),
  rendered on both the Home and Gallery pages — a responsive row of vertical "reel" cards with
  native controls, `muted` + `playsInline`.
- **To add more videos:** they must be **H.264** (not HEVC) to play everywhere. Check a `.mov`'s
  codec with `grep -a -o -m1 'avc1\|hvc1\|hev1' file.mov` (`avc1` = good). Then copy in as
  `public/videos/clip-NN.mp4` and add the path to the `CLIPS` array in `VideoShowcase.tsx`.
  Properly compressing the HEVC originals would need `ffmpeg` (not installed in this environment).
